Phase 3: Normalized Corrections (V9+) local_std Norm

Why Normalize Corrections?

V7 visual validation showed the model detects noise in the right areas but applies insufficient corrections. A cell needing a 70m correction only received 0.4m. The root cause: Huber loss with delta=1.0 gives identical gradients for any error above 1m, so the correction head cannot learn large magnitude corrections.

Raising delta alone did not help (V8). The 95th percentile of raw corrections is only 0.652m, so delta clamped to the min_delta floor of 1.0. The data is dominated by small corrections with extreme spikes in the tail.

V9 normalizes correction targets by local surface variability (local_std). A 70m spike in variable terrain (local_std=5m) becomes 14 std devs. A 0.3m spike in flat terrain (local_std=0.05m) becomes 6 std devs. Both are now comparable training signals. This also prepares the model for multi-location training data spanning different depth regimes.

Correction Analysis: Why V7 Under-corrects

The 70m Problem

V7 correctly identified noise locations but predicted a 0.4m correction where 70m was needed. The QGIS diff layer (diff_v7) confirmed large residual errors at cells with known noise.

The correction head's Huber loss (delta=1.0) gives identical gradient magnitude for any error above 1m. The model has no incentive to predict 70m over 2m because both errors produce the same training signal. Combined with the training data distribution (mean correction 0.315m, 95th percentile 0.652m), the model converges on predicting small corrections for everything.

V8 attempted to fix this by computing delta from the data's 95th percentile, but the 95th percentile (0.652m) was below the min_delta floor, resulting in delta=1.0 again.

V9 attacks the root cause: normalize corrections by local surface variability. After normalization, the Huber delta of 1.0 covers the same proportion of corrections, but the model now learns in relative units. At inference, predictions are multiplied back by local_std to produce meter-scale corrections. Values beyond 50 standard deviations are capped to prevent pathological training signals in flat areas where local_std is near zero.

Key Observations & Lessons

  • V1/V2 (Synthetic, 10 tiles): Both oscillated around 48% val accuracy, near random. Adding uncertainty (V2) gave negligible improvement. CPU-only, very limited data.
  • V3 (Ground truth, shoal loss, no weights): First model on real data. Too conservative at 0.6% noise detection.
  • V4 (Manual 2x noise weight): Overcorrected to 96.8% noise with low confidence (0.455).
  • V5 (No class weights bug): Collapsed to all-seafloor. 67% accuracy matched seafloor proportion exactly.
  • V6 (Auto-computed weights): Broke the all-seafloor pattern. Only 3.3% detection because features were contaminated at survey boundaries where nodata (1e6) bled into local stats via uniform_filter with mode='nearest'.
  • V7 (Boundary-aware features): Boundary fix used masked local stats (only valid neighbors). Noise detection jumped to 34.8% matching ground truth distributions. Peak val accuracy 72% exceeded seafloor proportion (75%), confirming genuine noise classification. Visual validation: correct spatial distribution but insufficient correction magnitude.
  • V8 (Dynamic Huber delta): Data-derived delta computation added. 95th percentile of raw corrections (0.652m) was below min_delta=1.0, producing identical training dynamics to V7. No inference run needed.
  • V9 (local_std normalization): Correction targets divided by per-node local_std, capped at +/-50 std devs. Huber delta computed from normalized distribution. Denormalization at inference multiplies by local_std to recover meters. Prepares the pipeline for multi-location training data.
  • Persistent overfitting: Val loss diverges after ~5 epochs across all ground truth versions. All 4 GT pairs from the same Seward area. Geographic diversity is the highest-priority improvement.
  • Missing feature -- sounding density: The strongest discriminator between noise and legitimate features is the number of soundings per cell. Single-hit cells are almost certainly noise; multi-hit cells are likely real. Not part of the BAG standard output. Could be computed from point cloud data as a preprocessing step in the future.
  • Classification accuracy plateau: 72% val accuracy with 75% seafloor proportion indicates the model is only marginally better than majority-class prediction. Both false positives (good seafloor flagged as noise) and false negatives (noise missed) are present. More training data diversity is expected to be the largest single factor in improving this.

Next Steps

  • Add geographic diversity: New matched survey pairs from other locations. Re-run prepare_ground_truth.py to generate additional GT files, then retrain as V10.
  • Validate V9: Run inference on Seward 1of4 and compare correction magnitudes against V7 diff layer. The denormalized corrections should produce larger values at high-noise cells.
  • Adaptive Huber delta (Options 2/3): Documented in losses.py. Option 2: per-epoch delta from prediction errors. Option 3: data-derived ceiling with adaptive decay.
  • Sounding density feature: Investigate computing per-cell sounding count from point cloud data as a preprocessing step.
